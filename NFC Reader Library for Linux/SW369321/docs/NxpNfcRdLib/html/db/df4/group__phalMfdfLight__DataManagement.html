Reads data from standard data files or backup data files.

Remarks
Chaining upto the size of the HAL Rx buffer is handled within this function. If more data is to be read, the user has to call this function again with bOption = PH_EXCHANGE_RXCHAINING | [one of the communication options]
Returns
Status code
Return values
PH_ERR_SUCCESSOperation successful.
PH_ERR_SUCCESS_CHAININGindicating more data to be read.
Dependingon implementation and underlying component.
Parameters
[in]pDataParamsPointer to this layer's parameter structure.
[in]bOptionCommunication settings for the file.
  • PHAL_MFDFLIGHT_COMMUNICATION_PLAIN
  • PHAL_MFDFLIGHT_COMMUNICATION_MACD
  • PHAL_MFDFLIGHT_COMMUNICATION_ENC
PH_EXCHANGE_RXCHAINING: To be Or'd with the above option flag if Chaining status is returned.
[in]bInsUses ISO 14443-4 chaining instead of DESFire application chaining. This should be always Set.
[in]bFileNoThe file number from where the data to be read.
[in]pOffsetThe offset from where the data should be read. Will be of 3 bytes with LSB first. If 0x10 need to be offset then it will be 10 00 00.
[in]pLengthThe number of bytes to be read. Will be of 3 bytes with LSB first. If 0x10 bytes need to be read then it will be 10 00 00.
If complete file need to be read then it will be 00 00 00.
[out]ppRxdataThe data retuned by the PICC.
[out]pRxdataLenLength of bytes available in RxData buffer.

§ phalMfdfLight_ReadRecords()

phStatus_t phalMfdfLight_ReadRecords ( void *  pDataParams,
uint8_t  bCommOption,
uint8_t  bIns,
uint8_t  bFileNo,
uint8_t *  pRecNo,
uint8_t *  pRecCount,
uint8_t *  pRecSize,
uint8_t **  ppRxdata,
uint16_t *  pRxdataLen 
)

Reads out a set of complete records from a Cyclic or Linear Record File.

Remarks
The readrecords command reads and stores data in the rxbuffer upto the rxbuffer size before returning to the user. The rxbuffer is configured during the HAL init and this is specified by the user.

Chaining upto the size of the HAL Rx buffer is handled within this function. If more data is to be read, the user has to call this function again with bCommOption = PH_EXCHANGE_RXCHAINING | [one of the communication options]

If TMI collection is ON, if pRecCount is zero then pRecSize is madatory parameter. If pRecSize and RecCount are zero and TMI collection is ON, then PH_ERR_INVALID_PARAMETER error returned. If TMI collection is ON; and if wrong pRecSize is provided, then wrong RecCount value will be calculated and updated for TMI collection.

Returns
Status code
Return values
PH_ERR_SUCCESSOperation successful.
PH_ERR_SUCCESS_CHAININGindicating more data to be read.
Dependingon implementation and underlying component.
Parameters
[in]pDataParamsPointer to this layer's parameter structure.
[in]bCommOptionCommunication settings for the file.
  • PHAL_MFDFLIGHT_COMMUNICATION_PLAIN
  • PHAL_MFDFLIGHT_COMMUNICATION_MACD
  • PHAL_MFDFLIGHT_COMMUNICATION_ENC
PH_EXCHANGE_RXCHAINING: To be Or'd with the above option flag if Chaining status is returned.
[in]bInsUses ISO 14443-4 chaining instead of DESFire application chaining. This should be always Set.
[in]bFileNoThe file number from where the data to be read.
[in]pRecNo3 bytes LSB first. Record number of the newest record targeted, starting to count from the latest record written.
[in]pRecCount3 bytes LSB first. Number of records to be read. If 0x00 00 00, then all the records are read.
[in]pRecSizeThe number of bytes to be read. Will be of 3 bytes with LSB first.
[out]ppRxdataThe data retuned by the PICC.
[out]pRxdataLenLength of bytes available in RxData buffer.
